## 一、为什么现在要谈"具身智能"

过去十年，AI 在数字世界里完成了一次接近"通用化"的飞跃：从单任务深度学习模型，到能写代码、做研究、跨模态推理的大模型。但站在 2026 年回头看，所有这些能力都有一个共同前提——**它们存在于纯数字的封闭环境里**：输入是文本/图像，输出还是文本/图像，没有物理世界的延迟、噪声、接触力、磨损与意外。

**具身智能（Embodied Intelligence / Embodied AI）** 想做的事情是：把这一代 AI 的认知能力，搬到一个具有"身体"的载体上，让它在物理世界里**感知—决策—执行—再感知**地闭环运行。具身的"身"不只是一具机械结构，而是一个不可回避的事实——智能体的能力、限制、学习方式都受身体约束。

这件事 60 年前的机器人学就在做。今天和过去的差别在于：

1. **算力**：单台 GPU 训练能力比 2010 年提升 ~1000×，仿真并行规模从单环境变成万环境
2. **数据**：互联网视频、Open-X、DROID、RH20T 等真机数据集出现了第一波"规模化迹象"
3. **模型**：基础模型（VLM、视频生成模型）给机器人提供了通用先验，不必从零学习每个物体、每个场景
4. **硬件**：人形机器人本体进入"百花齐放但价格仍高"的阶段，机械臂、灵巧手成本逐年下降
5. **资本**：2024–2025 年具身赛道融资规模与公司数量呈现非线性增长

这五个条件叠加，使"具身智能能做到接近通用"第一次成为一个**严肃的工程问题**，而不只是学术猜想。

## 二、概念辨析

容易混淆的几个词：

| 概念 | 关注什么 | 与具身智能的关系 |
| --- | --- | --- |
| **机器人学（Robotics）** | 物理实现：机构、控制、规划 | 具身智能的"地基" |
| **传统 AI / LLM** | 数字世界推理 | 具身智能的"大脑"组件之一 |
| **Agent**（数字智能体） | 在软件环境中调用工具完成任务 | 把它"放进物理世界"就更靠近具身智能 |
| **多模态大模型（VLM）** | 跨模态理解 | 具身智能感知与高层规划的常用骨干 |
| **具身智能** | 感知—决策—执行闭环、物理交互、世界模型 | = 机器人学 + 大模型 + 数据驱动学习 |

一个判断标准：**如果你的"智能"在拔掉电机和传感器后还能完整运行，它就不是具身的。**

## 三、发展脉络（粗线条）

### 1. 经典机器人时代（1960s – 2010s 初）
- 工业机械臂主导，控制精度优先
- 算法栈：运动学、动力学、PID、轨迹规划
- 任务是"已知世界 + 固定流程"的自动化
- 代表：PUMA、KUKA、ABB、Fanuc

### 2. 移动机器人与 SLAM 时代（2000s – 2010s）
- 概率机器人学（Thrun 等）成为范式
- ROS 出现，建立开源生态
- 代表事件：DARPA Urban / Robotics Challenge、Boston Dynamics 早期视频

### 3. 深度学习初步介入（2015 – 2020）
- 视觉感知模块被 CNN 替换
- 强化学习 + 仿真在 locomotion 上跑通（ANYmal、MIT Cheetah）
- 抓取学习（Dex-Net、GraspNet）走通"数据驱动操作"路线

### 4. 端到端策略 + 大规模数据（2021 – 2023）
- RT-1、RT-2 把"语言 + 视觉 → 动作"端到端串起来
- ALOHA、Mobile ALOHA 提出可负担的双臂遥操作平台
- Diffusion Policy 把生成模型搬进策略学习

### 5. VLA 与人形浪潮（2023 – 2025）
- Open-X Embodiment 推动跨本体数据池
- OpenVLA 开源，社区有了可二次开发的强 baseline
- π0 / π0.5 把 flow matching 范式带进 VLA
- 人形机器人创业潮（Figure、1X、Optimus、宇树、智元、星动…）

### 6. 世界模型与"想象式学习"（2024 – 当下）
- 视频生成模型（Sora 类）被重新解释为"通用世界模拟器"
- DayDreamer、UniSim、Genie 等给机器人提供"在想象中训练"的可能
- 真机数据 + 仿真 + 视频先验三位一体的训练 pipeline 成形

> 注：这是一个**简化的叙事**。真实历史是多线并进的（例如阻抗控制、IK 求解、SLAM 这些方向从未停止演进），后续章节会在各自领域里给出更精细的时间线。

## 四、技术全景图

把具身智能拆成 6 个层次：

```
┌─────────────────────────────────────────┐
│ L6  产品 / 商业 / 法规                    │ ← 第 7 章
├─────────────────────────────────────────┤
│ L5  策略层：RL / IL / VLA / WorldModel   │ ← 第 5 章
├─────────────────────────────────────────┤
│ L4  任务/运动规划，导航，操作              │ ← 第 4 章
├─────────────────────────────────────────┤
│ L3  状态估计、定位、SLAM                  │ ← 第 3 章
├─────────────────────────────────────────┤
│ L2  感知（2D/3D/多模态）                  │ ← 第 2 章
├─────────────────────────────────────────┤
│ L1  本体 / 控制 / ROS2                    │ ← 第 1 章
├─────────────────────────────────────────┤
│ L0  仿真平台、数据基础设施                 │ ← 第 6 章（横切）
└─────────────────────────────────────────┘
```

几点观察：

- **L1–L4 是机器人学传统强项**：理论成熟，工程化程度高，但泛化能力差
- **L5 是当前数据驱动主战场**：泛化潜力大，但稳定性、可靠性远不如 L1–L4
- **L0 越来越重要**：决定了上层迭代速度
- **L6 决定了项目能不能"活下去"**——技术再炫，跑不通商业闭环就停留在 demo

## 五、主要学派

按"信息从哪里来 / 决策如何形成"分类：

### 1. 模型驱动派（Model-Based）
- 解析建模 + 优化求解（MPC、WBC、QP）
- 优点：可证、可解释、对动力学准确的任务（如足式运动）效果极佳
- 缺点：依赖准确建模，对接触富、不可建模的任务无能为力
- 代表：Boston Dynamics（早期）、MIT Cheetah、ETH ANYmal

### 2. 数据驱动派（End-to-End Learning）
- 用大数据 + 大模型直接学策略
- 优点：可扩展、潜力大，能吃多模态先验
- 缺点：数据贵、可靠性差、推理慢、长尾失败模式不可控
- 代表：RT-2、OpenVLA、π0、GR-2

### 3. 混合派（Hybrid / Structured Learning）
- 把经典控制器与学习模块组合：残差 RL、可微仿真梯度、模型 + 学习的 MPC
- 优点：兼顾稳定与泛化
- 缺点：架构复杂、调试成本高
- 代表：HIL-SERL、Diffusion Policy + Residual MPC、可微 sim2real

### 4. 世界模型派（World-Model-Based）
- 学习"环境如何演化"，在想象中规划和训练
- 优点：sample efficient、能利用海量被动视频
- 缺点：世界模型本身仍不够准确，闭环误差累积
- 代表：DreamerV3、UniSim、1X World Model

短期看，**混合派工程上最实用；长期看，世界模型 + 大规模 VLA 的组合最可能逼近通用具身。**

## 六、当前状态（截至 2026 年中的个人观察）

### 已经基本跑通的
- 室内**移动机器人导航**（清洁、配送、巡检）
- 仓库 / 工厂场景的**结构化操作**（搬运、上下料、分拣）
- **四足 locomotion**：复杂地形行走、跑步、跳跃，sim2real 较成熟
- 简单**家庭场景的 demo**：擦桌、折衣、倒水（受限条件下）

### 接近临界点但不稳的
- **通用抓取与放置**：演示成功率 80–95%，长尾仍卡
- **VLA 在新场景的 zero-shot 操作**：受限 prompt 下 OK，开放任务仍弱
- **人形全身操作**：稳定行走 + 双臂操作的耦合还在工程化

### 尚远的
- 完全开放环境的**长程任务**（"帮我做一桌饭"）
- **可靠的人机协作**（在不可预测人类动作下保持安全）
- **跨本体真正通用的策略**（同一个 policy 跑不同本体）

## 七、关键开放问题

1. **数据**：真机数据扩展性差、仿真有 reality gap、视频缺少动作标签——三者目前都没有彻底解法
2. **评测**：缺乏统一、可复现、贴近真实的 benchmark；论文里 60% 成功率到部署可能是 0%
3. **泛化层级**：物体泛化 < 场景泛化 < 任务泛化 < 形态泛化，每层都是断崖
4. **推理速度 vs 模型容量**：高频控制需要 100–1000 Hz，大模型推理还在 5–30 Hz
5. **安全与失败模式**：学习策略的失败往往不可预测，工业部署需要兜底层
6. **形式化保证**：如何让学习策略满足安全/约束（CBF、shielded RL 等方向在推进）
7. **硬件成熟度**：灵巧手、触觉、低成本伺服仍是瓶颈
8. **商业闭环**：toC 难度极高，toB 场景定义比技术更关键

## 八、未来 3–5 年的几个猜想

- **大数据集会进一步整合**（类似 LAION 之于 SD），跨本体预训练成标配
- **VLA + 经典控制器的混合架构**会成为工程主流，而不是纯端到端
- **世界模型会先在仿真和数据合成里产生价值**，闭环规划在 2–3 年内逐步落地
- **人形会经历"硬件超前 + 软件追赶"周期**，2–3 年内出现少数能真正商用的细分场景
- **数据采集环节会产生独立赛道**：teleoperation 工具、动捕数据公司、数据标注/质检
- **评测会从 "demo 视频" 慢慢转向 "可复现 benchmark + 真实部署成功率"**——这件事一旦发生，研究节奏会被显著拉直

以上都是个人判断，半年后回来打脸。

## 九、本仓库的学习路径建议

| 阶段 | 周期 | 内容 |

每章建议产出物：**一份精读笔记 + 一份源码分析 + 一个 demo（视频或脚本）**。
